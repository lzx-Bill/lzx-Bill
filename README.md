## How I Work

```text
Understand the system before changing it
Define constraints before implementation
Break complex goals into verifiable tasks
Keep execution state visible
Validate outputs instead of trusting them
Commit changes intentionally
```

我偏好的开发流程：

1. 扫描现有系统和约束
2. 明确目标、边界与验收标准
3. 拆分可验证任务
4. 按顺序执行并记录状态
5. 运行测试、检查 diff
6. 形成清晰的提交记录
7. 总结风险和后续事项
